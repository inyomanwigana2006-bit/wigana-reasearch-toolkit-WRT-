# Wigana Research Toolkit (WRT)

Wigana Research Toolkit (WRT) is a modern, web-based analytical suite designed to assist researchers in importing data, cleaning datasets, converting scales, conducting reliability and validity tests, and automatically generating academic interpretations and Chapter IV (Bab IV) write-ups.

## 🚀 Key Features

1. **Data Import & SPSS Reader**: Import Excel/CSV data or scan/read SPSS outputs with OCR and analysis capability.
2. **Smart Scale Converter**: Automatically standardize scale measurements across different surveys or questionnaires.
3. **Smart Cleaning**: Auto-detect anomalies, outliers, duplicate cases, and missing data points, with built-in imputation.
4. **Questionnaire Construct Builder**: Formulate constructs, map items to latent variables, and compute composite scores.
5. **Interactive Data/Variable View**: An SPSS-like spreadsheet interface to view, edit, and inspect data values and metadata in real-time.
6. **AI Analytical Interpretation**: In-depth, automated statistical synthesis and academic write-up recommendations for your outputs.
7. **Auto Bab IV Generator**: Instantly compile descriptive statistics, assumptions, reliability/validity tests, and hypothesis testing outputs into formal academic reports.

## 📦 How to Run Locally

Since WRT is built using pure client-side web technologies (HTML5, Javascript, CSS via Tailwind CSS, and Chart.js), it does **not** require any backend server, compilation, or complex installation.

1. Clone or download this repository:
   ```bash
   git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
   ```
2. Double-click the `index.html` file in the root directory, or open it with any modern web browser (Chrome, Firefox, Edge, Safari).
3. Alternatively, you can serve it using a lightweight local web server (e.g., Live Server in VS Code, or `npx serve`).

## 🌐 How to Deploy to GitHub Pages

You can host and run the Wigana Research Toolkit online for free using **GitHub Pages**:

1. Create a new repository on your GitHub account.
2. Push all the files in this directory to your new repository.
3. Go to the repository **Settings** on GitHub.
4. Click on **Pages** in the left sidebar under the "Code and automation" section.
5. Under **Build and deployment**, select **Deploy from a branch** as the source.
6. Choose the branch (e.g., `main` or `master`) and folder (select `/ (root)`).
7. Click **Save**.
8. Within 1-2 minutes, GitHub will provide a public link (e.g., `https://your-username.github.io/your-repo-name/`) where your application will be live!
